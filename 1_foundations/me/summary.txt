## Work Philosophy & Testing Approach

My philosophy on testing and quality centers on Behavior-Driven Development (BDD) and shifting testing to the left of the SDLC. Rather than having QA teams do all the testing after development is complete, I believe in embedding quality practices throughout the entire software development lifecycle. This means developers write tests alongside code, using BDD frameworks to ensure requirements are clearly understood and testable from the start. By shifting testing left, we catch issues earlier, reduce rework, improve collaboration between developers and QA, and ultimately deliver higher quality software faster. This philosophy drives my work in building automation frameworks and tools that enable teams to adopt these practices at scale.

## Achievements & Recognition

I was a co-inventor on this patent for an innovative log analysis system that uses ensemble machine learning models to detect anomalies in test execution logs. The system applies multiple prediction models to identify anomalous subsequences and automatically determines next best test cases, significantly improving test failure analysis and remediation efficiency. Patent link: https://patents.google.com/patent/US11513927B1/

**US Patent US11934302B2** - Machine Learning Method to Rediscover Failure Scenario by Comparing Customer's Server Incident Logs with Internal Test Case Logs (Granted March 2024). I was a co-inventor on this patent for a method that matches customer server incident logs (e.g., lifecycle controller logs) to internal test-case logs in a knowledge base. The pipeline uses Sequence Graph Transformation (SGT) to encode message-code sequences and capture short- and long-term dependencies between codes, classifies message codes into Critical, Warning, and Informational categories, applies PCA for dimensionality reduction, and then uses cosine similarity with category-weighted scoring to find the best-matching internal test case. The system recommends the corresponding solution and steps to reproduce the failure, enabling support engineers to resolve field issues faster. Patent link: https://patents.google.com/patent/US11934302B2
